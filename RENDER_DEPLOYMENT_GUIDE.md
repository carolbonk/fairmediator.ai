## 🔧 Troubleshooting Common Issues

### Issue: "Application failed to respond"

**Cause**: App not listening on PORT environment variable

**Fix**: Server.js already handles this:
```javascript
const PORT = process.env.PORT || 4001;
```

### Issue: "CORS error in browser"

**Cause**: CORS_ORIGIN doesn't include Render URL

**Fix**: Update CORS_ORIGIN in Render dashboard:
```
CORS_ORIGIN=https://fairmediator.ai,https://fairmediator-api.onrender.com
```

### Issue: "Database connection failed"

**Cause**: MONGODB_URI not set or incorrect

**Fix**:
1. Get connection string from MongoDB Atlas
2. Add to Render environment variables
3. Redeploy

### Issue: "Cron jobs not running"

**Check**: Look for this in server.js logs:
```
// Only start cron jobs in production AND non-serverless environments
if (process.env.NODE_ENV === 'production' && !isServerless)
```

**Solution**: Cron jobs should work on Render (not serverless). Check logs for "Daily scraping started".

### Issue: "Cold starts (first request slow)"

**On Render Free Tier**: Service spins down after 15 minutes of inactivity

**Solutions**:
- Upgrade to Starter plan ($7/month for always-on)
- Use cron-job.org to ping `/health` every 10 minutes
- Accept 30-second first-request delay as free tier trade-off
